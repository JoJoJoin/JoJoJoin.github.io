Godot4 打包

发表于 2026-03-31 分类于 技术

概述

Godot 4 提供了将项目导出为可分发包的功能，支持 Windows、Linux、macOS、Android、iOS、Web 等多个平台。本文记录 Godot 4 的打包流程和常见配置。

前置准备

安装导出模板（Export Templates）

导出模板是 Godot 为每个平台预编译的引擎二进制文件，打包时必须先安装。

  1. 打开 Godot 编辑器
  2. 菜单栏 → 编辑器（Editor） → 管理导出模板（Manage Export Templates）
  3. 点击 下载并安装（Download and Install），等待下载完成

模板版本必须与编辑器版本完全一致。如果网络不好，可以从 Godot 官网 手动下载 tpz 文件，然后点击 从文件安装（Install from File）。

Windows 平台打包

1. 添加导出预设

  1. 菜单栏 → 项目（Project） → 导出（Export）
  2. 点击 添加（Add） → 选择 Windows Desktop

2. 基本配置

配置项 说明
导出路径（Export Path） 输出的 .exe 文件路径
架构（Architecture） x86_64（默认 64 位）或 x86_32
嵌入 PCK（Embed PCK） 勾选后将资源包嵌入 exe，生成单文件

3. 执行导出

  • 导出项目（Export Project）：生成发布版本（Release），无调试符号
  • 导出 PCK/ZIP：仅导出资源包，不包含引擎可执行文件

4. 关于 rcedit（可选）

如果需要自定义 exe 的图标和版本信息：

  1. 下载 rcedit
  2. 在 编辑器设置（Editor Settings） → 导出（Export） → Windows 中设置 rcedit 路径
  3. 在导出预设中配置 应用程序（Application） 下的图标和版本号

Linux 平台打包

  1. 添加导出预设 → 选择 Linux
  2. 配置导出路径为 .x86_64 文件
  3. 导出即可

生成的文件包含：

  • 游戏名.x86_64：可执行文件
  • 游戏名.pck：资源包（除非勾选嵌入 PCK）

macOS 平台打包

  1. 添加导出预设 → 选择 macOS
  2. 导出路径设为 .dmg 或 .zip
  3. 如需公证（Notarization），需要配置 Apple Developer 证书信息

macOS 导出可在任意平台进行，但签名和公证需要 macOS 环境或配置远程签名。

Android 平台打包

前置要求

  1. 安装 Android SDK（推荐通过 Android Studio 安装）
  2. 安装 JDK 17
  3. 创建调试密钥库（Debug Keystore）：

配置编辑器

在 编辑器设置 → 导出 → Android 中填写：

  • Android SDK 路径
  • Debug Keystore 路径

导出

  1. 添加导出预设 → 选择 Android
  2. 配置包名（如 com.example.mygame）
  3. 最低 SDK 版本建议设为 24（Android 7.0）
  4. 目标 SDK 版本建议设为 33 或更高
  5. 导出为 .apk（直接安装）或 .aab（上架 Google Play）

Web 平台打包

  1. 添加导出预设 → 选择 Web
  2. 导出路径设为 .html 文件
  3. 导出后会生成多个文件（.html、.js、.wasm、.pck）

Web 导出需要通过 HTTP 服务器访问，直接打开 HTML 文件会因 CORS 限制无法运行。可用 python -m http.server 快速启动本地服务器测试。

常用导出选项

资源（Resources）

选项 说明
导出模式 导出所有资源 / 仅导出选中场景的资源
过滤器（Filters） 按通配符包含或排除特定文件（如 *.json, *.csv）

功能（Features）

可以设置自定义功能标签，在代码中通过 OS.has_feature("tag") 判断，用于区分不同的发布版本。

脚本加密

Godot 4 支持对导出的 GDScript 进行加密：

  1. 在导出预设中启用 脚本加密（Script Encryption）
  2. 设置 256 位 AES 密钥

常见问题

导出后运行闪退

  • 检查是否遗漏了资源文件（在过滤器中添加需要的文件类型）
  • 检查是否有使用 @tool 脚本在编辑器中正常但导出后出错
  • 查看日志：Windows 下运行 游戏.exe --verbose 或检查 user://logs/godot.log

导出体积过大

  • 开启 编辑器设置 → 文件系统（Filesystem） → 导入（Import） 中的纹理压缩选项
  • 排除未使用的资源
  • 对于 Web 平台，考虑使用 .zip 而非 .pck 以利用 HTTP 压缩

Android 导出报错 “No export template found”

确认导出模板版本与编辑器版本一致，且已正确安装 Android 导出模板。
